Точки входа:
1. класс Main - игра с сохранением статистики, сохраняет сразу 2 файла
(удалив один из объектов writer можно писать в один)

2. в папке Parsers класс ParserMain - спрашиваем какой тип и какой из файлов парсить

В классе Main

Файлы сохранения хода игры пишем в формате gameplay_dd_MM_yyyy_HH_mm_ss.xml/json чтобы не было одинаковых имен

Классы парсеров в папке parsers, кушаем разные виды форматов ввода координат хода игрока

классы моделей в папке models (Person) Для сериализации статистики игр,
остальные для создания модели при сериализации в json/xml

Все игровые методы находятся в классе GameMethods, его наследниками являются классы:

1 - GameWithHuman -реализует все игровые методы при игре с человеком,
результаты сохраняются в объект Gameplay

2. GameFromGameplay - реализует игровые методы при воспроизведении игры из объекта gameplay получаемого из json/xml
