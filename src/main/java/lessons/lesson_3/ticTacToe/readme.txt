Точки входа:
1. класс Main - игра с сохранением статистики
2. в папке Parsers класс StaxParser - парсинг файла XML и вывод хода игры на экран

Файлы сохранения хода игры пишем в формате gameplay_dd_MM_yyyy_HH_mm_ss.xml чтобы не было одинаковых имен

Классы парсеров (sax и stax) в папке parsers, кушаем разные виды форматов ввода координат хода игрока

классы моделей в папке models

Все игровые методы находятся в классе GameMethods, его наследниками являются классы:

1. GameWithHuman -реализует все игровые методы при игре с человеком
в нем же происходит сохранение результатов игры в XML файл, хотя с точки зрения ООП правильнее сохранять результаты в объект Gameplay и сделать класс Writer для записи объекта в XML файл
2. GameFromXml - реализует игровые методы при воспроизведении игры из XML
